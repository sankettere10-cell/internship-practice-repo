Software Testing is the process of verifying and validating a software application to ensure it meets requirements and works correctly without defects.

1. What is Software Testing?

Software Testing is used to:

Find defects (bugs)
Verify requirements
Improve software quality
Ensure reliability and performance
Reduce business risks
Example

If a login page should allow valid users to log in:

Enter correct credentials → Login successful ✅
Enter wrong credentials → Error message displayed ✅

Testing verifies both scenarios.

4. Software Testing Principles
1. Testing Shows Presence of Defects

Testing finds bugs but cannot prove software is bug-free.

2. Exhaustive Testing is Impossible

You cannot test every possible input.

3. Early Testing

Start testing as early as possible.

4. Defect Clustering

Most defects are found in a few modules.

5. Pesticide Paradox

Repeated test cases eventually stop finding new bugs.

6. Testing is Context Dependent

Testing approach varies by application.

7. Absence of Errors Fallacy

Bug-free software is useless if it doesn't meet user needs.

5. Types of Testing
Manual Testing

Testing without automation tools.

Advantages
Human observation
Better exploratory testing
Disadvantages
Time-consuming
Repetitive work
Automation Testing

Testing using scripts and tools.

Advantages
Faster execution
Reusable scripts
Suitable for regression
Disadvantages
Initial setup cost
Requires programming knowledge
6. Levels of Testing
Unit Testing

Performed by developers.

Example:

int sum(int a, int b)

Testing the function independently.

Integration Testing

Testing interaction between modules.

Example:

Login Module → Dashboard Module
System Testing

Testing the complete application.

Acceptance Testing

Performed by clients/users.

Types
Alpha Testing
Beta Testing
7. Functional Testing

Checks whether functionality works according to requirements.

Examples
Login
Registration
Search
Payment
8. Non-Functional Testing

Checks quality attributes.

Types
Performance Testing
Security Testing
Usability Testing
Compatibility Testing
Reliability Testing
9. Smoke Testing

Checks critical functionalities after a new build.

Example
Application launches
Login works
Dashboard opens

If smoke testing fails, build is rejected.

10. Sanity Testing

Checks a specific functionality after bug fixes.

Example

Bug fixed in login page.

Only login-related features are tested.

11. Regression Testing

Ensures new changes do not affect existing functionality.

Example

Developer modifies payment feature.

Tester verifies:

Payment
Login
Search
Cart

still work correctly.

12. Retesting

Testing the same defect after developer fixes it.

13. Exploratory Testing

Testing without predefined test cases.

Tester explores application behavior.

14. Ad-Hoc Testing

Informal testing without documentation.

15. Black Box Testing

Tester does not know internal code.

Focus:

Inputs
Outputs
16. White Box Testing

Tester knows internal code structure.

Focus:

Code coverage
Logic paths
17. Grey Box Testing

Combination of Black Box and White Box.
